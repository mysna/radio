# EPG 현재 재생 정보 디자인

## 목표

`radio_epg`의 `/v1/now` 배치 API를 사용해 현재 프로그램명·이미지·방송 시간 진행률을 현재 재생 영역과 재생목록에 표시하고, 같은 정보를 iPhone Media Session 잠금화면에도 반영한다.

## 설계

- API 기본 URL은 `src/config.js`의 설정값으로 둔다.
- 재생목록의 radio ID를 한 번에 `/v1/now?radio_ids=...`로 요청한다. API가 제공하는 최대 100개 배치와 30초 캐시를 활용한다.
- 응답은 채널 ID별로 메모리에 보관한다. 현재 이벤트의 `ends_at` 직전까지는 1초 타이머로 진행률만 갱신하고, 종료 후 한 번만 API를 재조회한다. 안전장치로 30초마다 갱신한다.
- 채널 변경, 페이지 가시성 복귀, 네트워크 재연결 시 즉시 갱신하며, 실패하면 기존 채널 UI를 유지하고 EPG 영역만 비운다.
- API의 상대 이미지 URL은 API 기본 URL과 결합한다.
- MediaMetadata는 `title=프로그램명`, `artist=채널명`, `artwork=프로그램 이미지`로 설정하고 프로그램이 없으면 기존 채널명 메타데이터로 돌아간다.

## UI

- 현재 재생 카드에 정사각형 커버 이미지, 프로그램명, 채널 메타데이터, 시간 진행률을 추가한다.
- 재생목록 각 행에 채널명 아래 현재 프로그램명을 추가한다.

## 검증

- EPG 응답 정규화·진행률·갱신 지연 계산을 순수 함수로 테스트한다.
- DOM 정적 테스트로 현재 재생/재생목록 필드와 설정값을 잠근다.
- 기존 `npm test`와 `node --check`를 실행한다.
